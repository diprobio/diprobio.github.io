# Shared Docs Folder

Use this folder for organization-level documents that are not tied to a single project repository.

Examples:

- team handbook
- meeting templates
- architecture overviews
- onboarding docs

Suggested structure:

- `docs/handbook/`
- `docs/templates/`
- `docs/onboarding/`

If a document belongs to one project, prefer storing it inside that project repository and linking it in `data/projects.json`.
